Title: бездомные веб компоненты (Light DOM web components)
======================================
https://lit.dev/
	LIT Web Components

https://github.com/illicchpv/wcc-father
	WCC Web Components 
	
в процессе изучения, мне понравилась модель Web Components.
и вот был создан базовый класс, на основании которого построение компонента должно быть проще. 
и в плане структуры веб компонент будет упорядочен.
Выбрана модель Light DOM без инкапсуляции стилей, 
Это предполагает использование какого-то метода уникализации стилей. например БЭМ.

почему не чистые компоненты - на мой взгляд, слишком много кода и приходится каждый раз придумывать свою структуру.
почему не Lit 
	- мне не нравится Shadow DOM. Bootstrap, Tailwind, свои CSS-файлы не проникают внутрь компонента. И приходится много делать внутри.
	- Кроме того, они требуют сборки. Без npm не очень понятно как их создавать. 
  - Ориентированы на сложные универсальные компоненты.

но это моё личное мнение 

в итоге был создан BaseComponent и сопутствующие файлы для реализации задумки.
далее:

wcc - префикс выбранный мной, для такого типа компонентов т.е. наследуемых от BaseComponent

что предлагается - 
	- разработка на нативных стандартах браузера
	- BaseComponent + Light DOM - Стили сайта работают сразу
	- разделение HTML JS
	- Автоматизация через соглашения
	- в HTML Используются HTML-комментарии для логики
		- Условия: <!-- if(this.isAdmin) --> ... <!-- endif -->
		- Списки: <!-- innerTemplate:user-card --> для повторения элементов.
		- Интерполяция: в <!-- include --> ${this.userName}.
		- Эмуляция слотов : Реализовали поведение <slot> для Light DOM
	- заранее созданная структура js части компонента
	- gitHub репозиторий для начала проекта.
	- набор сниппетов для облегчения создания компонентов
	
в результате - 
	- бездомные веб компоненты или Light DOM компоненты
	- Zero Dependencies : Работает без npm install , прямо в браузере (ES Modules).
	- Прозрачность : Это просто HTML и JS. Легко отлаживать, легко понимать.
	- можно использовать просто как инструмент построения html
